# TapLog — 製造日報デジタル化アプリ

自動プレス加工職場の製造日報（紙）をWebアプリ化した試作実装です。要件整理
[`TapLog_要件整理_v0.7.md`](./TapLog_要件整理_v0.7.md) と、同梱の画面イメージ
（`*_screen.html` / `admin_*.html`）に沿って構築しています。

UIは **Google Material Design 3** を参考にしたデザイントークンで構成し、
レスポンシブ対応。作業者画面はスマホ前提で**大きめの文字・タッチターゲット**に
しています。

## 技術構成

- **ビルド不要のクライアントSPA**（Vanilla JS / ES Modules）
- **オフラインファースト**：データは端末の `localStorage` に保存（要件 §9）。
  通信状態に依存せず動作し、画面右上にオンライン／オフラインを表示。
- 外部ランタイム依存なし（フォント・アイコンのみGoogle Fonts CDN）。

## 起動方法

ES Modules を使うため、ローカルHTTPサーバ経由で開きます（`file://` 不可）。

```bash
cd app
python -m http.server 8765
# ブラウザで http://localhost:8765/ を開く
```

入口で「作業者（スマホ）」「管理者（PC）」を選択します。

## 画面と要件の対応

### 作業者向け（スマホ・記録特化 / 要件 §7）
| 画面 | 内容 |
|------|------|
| ログイン | 社員証QRスキャン（シミュレート）／社員番号の手入力。オフライン動作。自動ログアウトなし。 |
| ホーム | 担当号機をカード表示。進行中ジョブの状態・経過を表示。「現在ログイン中」＋「交代」を常時表示。 |
| 新規ジョブ | 号機＋品番を選択 → 標準SPMを自動表示 → 開始。 |
| 記録 | 種類タップで開始／終了（時刻自動）。ライブのストップウォッチ。段取り計・検査計・負荷を自動集計。 |
| 生産実績 | 生産数／ロット／総ショット数／ロス、不良モード別個数（マスタ連動）。不良数を自動合計。 |

### 管理者向け（PC Web / 要件 §8）
| 画面 | 内容 |
|------|------|
| ホーム | 稼働サマリーKPI＋マスタ管理タイル。 |
| マスタ管理 | 品番／号機／ユーザー／不良モードの追加・編集・削除・有効無効・表示順。 |
| ダッシュボード | 主指標＝号機別・品番別稼働率。補助＝全体稼働率・平均段取り・不良率・総ショット数。期間切替（今日／今週／今月）。 |
| ジョブ一覧・詳細 | 一覧、詳細（自動集計・標準比較・イベント履歴）、CSVエクスポート。 |

## 自動計算（要件 §6）
- 不良数＝工程内不良（モード別個数）の合計
- 段取り計／検査計／その他＝同種イベントの所要時間を合算
- 負荷時間＝ジョブ開始〜終了の総経過
- 標準比較＝標準段取り時間・標準SPMと実績の差異・達成率

## ディレクトリ構成

```
app/
  index.html
  css/      tokens.css base.css worker.css admin.css
  js/
    lib/      id, dom, time, router, ui
    data/     seed, store（localStorage・不変更新）
    domain/   events（イベント種別）, calc（自動計算）
    worker/   login, home, job-new, record, result, shell
    admin/    home, masters, dashboard, jobs, export, shell
    launcher.js  main.js
```

## デモデータ
初回起動時に号機・品番・ユーザー・不良モードのマスタとサンプルジョブを投入します。
入口画面の「デモデータを初期化」でいつでもリセットできます。

## Vercel へのデプロイ

Next.js は不要です。**ビルドなしの静的サイト**としてそのままデプロイできます。
ハッシュルーティングのためSPAリライト設定も不要です。配信ルートは `app/` です。

### Vercel CLI（最短）
```bash
npm i -g vercel
cd app
vercel          # プレビュー
vercel --prod   # 本番公開
```

### GitHub 連携
1. このリポジトリを GitHub に push
2. Vercel で Import し、プロジェクト設定を以下に:
   - **Framework Preset**: Other
   - **Root Directory**: `app`
   - **Build Command** / **Output Directory**: 空欄

> フォント・アイコンは Google Fonts CDN を利用するため、オンライン環境で表示されます。
> データは各ブラウザの `localStorage` に保存され、端末ごとに独立します。
